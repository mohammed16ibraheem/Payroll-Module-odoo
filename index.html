1. What Was Wrong (BEFORE)

Bug 1: Vacation Cutting Salary

RAM LAL's payslip had Paid Time Off (12 days, 96 hours, 476.80 SR) pulled automatically from the Time Off module. This was reducing his salary even when we did not want it to.

Bug 2: Wrong Month in Title

The payslip period ended in December, but the title showed "November 2025 Payslip" because it used the start date instead of the end date.

2. Fix 1 — Stop Vacation from Auto-Cutting Salary

Where: Payroll → Configuration → Work Entry Types

We opened each leave type and unchecked the "Time Off" box. This stops vacation from automatically going into the payslip.

3. Fix 2 — Correct Month in Payslip Title

Where: Settings → Technical → User Interface → Views → Search report_payslip

We changed Line 4 of the report so it uses the end date of the period instead of the start date.

7. Complete Implementation Summary

Original Task Requirements

Payslips: Disable the relation between Time Off Module and Payroll Module so that vacation does not affect the salary, because we are making manual adjustment in salary for vacation or absences.

Note: Need to find other solution.

Payslip Report: Giving Payslip Heading as the previous month (i.e. taking from Start date of the period in Payslip dates). It should take End date month of the payslip period as Month Heading.

Understanding

Currently, payslips are linked to the Time Off module, so validated leaves/vacations automatically reduce worked days or salary. You now handle vacations/absences by manual adjustment lines in the salary rule, so Time Off should not change salary at all.

In the report, if the payslip period is 26-01-2026 to 25-02-2026, the heading now shows January, but it should show February (based on the period end date).

Implementation Steps (Where to Click)

Check Before: Payroll → Configuration → Settings. Open one payslip with vacations. Note Worked Days tab and report month heading.

Fix 1: Payroll → Configuration → Work Entry Types. For each leave type (Generic Time Off, Compensatory, Unpaid, Sick, Paid Time Off), uncheck "Time Off" under TIME OFF OPTIONS. Save.

Fix 2: Settings → Technical → User Interface → Views. Search report_payslip. Open hr_payroll.report_payslip. In Architecture tab, change Line 4 from t-field="o.name" to t-esc="o.date_to.strftime('%B %Y') + ' Payslip'". Save.

BEFORE — What Was Found

Bug #1: RAM LAL's payslip (Period: 21/11/2025 - 20/12/2025) had Paid Time Off (12 days, 96 hours, 476.80 SR) automatically pulled from Time Off module, reducing salary.

Bug #2: The same payslip had period ending December but the title showed "November 2025 Payslip" (using Start Date month instead of End Date month).

FIX 1 — Disable Time Off → Payroll Link

Where: Payroll → Configuration → Work Entry Types

Change: Uncheck "Time Off" for Generic Time Off (LEAVE100), Compensatory (LEAVE105), Unpaid (LEAVE90), Sick Time Off (LEAVE110), Paid Time Off (LEAVE120).

Result: Time Off checkbox UNCHECKED. Validated leave approvals no longer auto-import into payslip Worked Days. Manual salary adjustments are the only way vacation affects salary.

FIX 2 — Payslip Report Month Heading

Where: Settings → Technical → User Interface → Views → Search report_payslip → Open hr_payroll.report_payslip

BEFORE (Line 4): <h2 id="payslip_name"><span t-field="o.name">August 2023 Payslip</span></h2>

AFTER (Line 4): <h2 id="payslip_name"><span t-esc="o.date_to.strftime('%B %Y') + ' Payslip'"/></h2>

  2. Future Odoo module updates may overwrite the QWeb report change. Re-apply Line 4 or use an Inherited View via Odoo Studio for a permanent solution.
  3. Alternative: Keep Time Off link enabled but set leave Work Entry Types to zero amount/coefficient in Salary Structure rules — leaves tracked for reporting but don't reduce salary.
